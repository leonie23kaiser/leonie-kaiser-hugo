# hugo-add-images

Add and organize images in the Hugo asset tree.

## Triggers
"add image", "upload image", "organize images", "banner photo", "fighter portrait", "OG image"

## When to Use
- Placing a new image into the correct folder under `assets/images/` or `static/images/`
- Naming and sizing images to meet site conventions
- Adding OG images, fighter portraits, or event banners
- Does NOT generate images — for AI generation use `hugo-create-image`

## What to Gather

1. Image file or source path.
2. Purpose: fighter portrait, event banner, hero, OG image.
3. Target content type (fighters, events, homepage).

## Folder Structure

```
src/superleague.tv/
  assets/images/          # pipeline-processed images (WebP srcset via picture.html)
    fighters/             # portrait + background images per fighter
    events/               # fight night banners
    hero.jpg              # homepage hero
    logo.png
  static/images/
    fighters/             # static portrait fallbacks (not pipeline-processed)
```

**Key rule:** Images must be in `assets/` (not `static/`) to go through the Hugo pipeline (`picture.html` partial → WebP + srcset). Static images are unprocessed fallbacks only.

## Image Sizing Guide

| Usage | Size | Format | Notes |
|-------|------|--------|-------|
| Fighter portrait | 400×500px | JPG/PNG | 4:5 ratio, `object-fit: cover` |
| Fighter background | 1200×600px | JPG | full-bleed hero on single page |
| Event banner | 1200×630px | JPG | used in cards and OG |
| Homepage hero | 1920×1080px | JPG | LCP image, use `eager: true` |
| OG image | 1200×630px | JPG | social sharing |

## Format Guidelines
- **JPG**: photographs, portraits, banners (smaller file size)
- **PNG**: graphics with transparency
- **WebP**: auto-generated by Hugo pipeline — do not add manually

## Referencing in Front Matter

```yaml
# Fighter
portrait: "images/fighters/first-last.jpg"   # path under assets/

# Event
banner: "images/events/fight-night-name.jpg"
```

## Workflow

1. Identify the image purpose and content type.
2. Place the image in the correct `assets/images/` subfolder.
3. Name it using the content slug (e.g., `valon-basha.jpg`).
4. Add the `assets/`-relative path to the target frontmatter field.
5. Verify display and responsive behavior via `picture.html` partial.
